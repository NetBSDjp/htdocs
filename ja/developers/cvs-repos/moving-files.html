Moving and copying files in the CVS repository

Problem

Now and then a situation occurs in which it is necessary to move or copy files in the CVS repository and keep the CVS history. This is not supported by CVS itself, so it can't be done using CVS's normal client-server mechanisms.

To be able to do this properly and consistently, a couple of small scripts have been written that can do these things "the correct way". The scripts are located on the master CVS server(s) in /usr/local/bin, and are named:

  • cvscopy
  • cvsmove
  • deadify
The cvscopy and cvsmove scripts take two arguments; the from-file-name and the to-file-name. Note that you must give both names to the command! Deadify takes one file name and declares all previous revisions as dead.

Note that you must be logged in on cvs.netbsd.org and are playing with the master CVS repository, so be careful!

FAQ

Q: What does it mean if cvsmove or cvscopy asks me:

  • Owerwrite ......
A: It means that you have alias rm rm -i. You should answer "y" to this question.
