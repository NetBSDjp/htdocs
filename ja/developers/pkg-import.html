新しいパッケージのインポート

インポートしようとしているパッケージが正しい (pkglintしたり、テストしたりなど) ことをいったん確かめたら、以下の手順を使ってください: (一連の"cvs add"コマンドを使用しないでください)

  1. インポートする前にパッケージをテストするため、"pkglint -a -v"を使用してください。 致命的なエラーがあってはいけませんし、警告もあるべきではありません。 これらのメッセージの中で意見が合わないものがありましたら、 "packages@netbsd.org"まで連絡をください。

  2. "pkgsrc/Packages.txt"の第8章にいくつかデバッグの手法が書かれています。 特に、"find"テストはパッケージリストを確実に完成させるためにも使用すべきです。

  3. もしあなたがコミットしたパッケージがFreeBSDの"ポート"に基づいているならば、 以下のものを使ってください:

    cvs import pkgsrc/category/pkg FREEBSD FreeBSD-current-1998-09-28

    (日付、パッケージのカテゴリー、名前がパッケージを表すところ) 。
    FreeBSDの"ポート"を変えずにインポートしNetBSDで動くようにするために必要な変更をコミットするべきです。

  4. もしパッケージが新しいならば、かわりに以下のものを使ってください:

    cvs import pkgsrc/category/pkg TNF pkgsrc-base

    (カテゴリーとパッケージ名が再びパッケージを表すところ) 。

  5. コミットメッセージにはパッケージの名前とバージョン番号、"pkg/COMMENT"の内容が含まれるべきです。
注意: pkgsrc/category/pkgパスはcvsrootからの相対パスで、 あなたがインポートしているファイル用のパスではありません。 インポートするときは、パッケージのトップレベルのディレクトリーにいなければいけません!
