NetBSD/sparc64

NetBSD/sparc64 について 

NetBSD/sparc64 は Sun の 64ビット UltraSPARC 機 への NetBSD のポートです。 これは、 Sbus ベースの UltraSPARC システム (Ultra1 と Ultra2)をサポートし、 UltraSPARC IIi CPU (Ultra5, Ultra10 と SPARCengine AXi) ベースで PCI ベース の UltraSPARC システムの多くをサポートしています。 UltraSPARC I & II PCI システム では最近初めてマルチユーザー起動が行われました が、これまで Enterprise 250 システム上だけでテストされています。

NetBSD/sparc ポートのほかのドライバーは 一度移植されれば動くようになるでしょう。

現在、 NetBSD/sparc64 は一定の Ultra プラットホームをサポートしています (下を見てください)。 32ビットカーネル は ほとんどの NetBSD/sparc バイナリー を実行します。 (カーネル精査ツールは例外で、修正される事を期待します)。 64ビットカーネル では 64ビットバイナリーが走り、 そのうえ、32ビット バイナリー を走らせるサポートも行われています。 (だけど、これはまだ開発中です).

簡単にインストールできる NetBSD/sparc64 のバイナリー配布物は 最新の 1.5 リリース と、開発トランク(幹)のスナップショットがあります。


NetBSD/sparc64 News 

2001-05-19: 1.5.1_BETA2 スナップショットができました。
NetBSD/sparc64 1.5.1_BETA2 スナップショット は 2001-05-18 の 1.5.1 ソース に基づいていて、 NetBSD/sparc64 ftp サーバーの スナップショット置き場 にあります。

2001-02-26: NetBSD/sparc64 LKM が動きます
Matthew Green は NetBSD/sparc64 ポートの LKM を改修し、 <sys/lkm.h> の小さな変更を コミット しました。 これを利用するには、カーネルと modload(1) を、修正された <sys/lkm.h> を使って再構築する必要があります。 古いカーネルで LKM をロードしようとするとパニックし、 古い modload(1) を使うと ENOTTY (デバイスへの不適切な ioctl )が帰ってくるでしょう。

Update: これは netbsd-1-5 ブランチに組み込まれたので、 NetBSD 1.5.1 には入るでしょう。

2001-02-10: 新しい COMPAT_SVR4_32 オプション
Eduardo Horvath が COMPAT_SVR4_32 オプションを NetBSD-current に追加しました。 これをカーネルコンフィグレーションファイルに含めることで、 NetBSD/sparc64 で 32ビット SVR4 バイナリーの互換機能が得られます。

2001-02-04: 新しい netbsd32_compat14 と netbsd32_compat15 パッケージ
Matthew Green は netbsd32_compat14 と netbsd32_compat15 パッケージを pkgsrc に追加しました。 この2つのパッケージ には 32ビット NetBSD/sparc a.out 共有ライブラリー(netbsd32_compat14) と 32ビット elf 共有ライブラリー(netbsd32_compat15) が含まれています。 これによって、 NetBSD/sparc64 で、 NetBSD/sparc のもとでコンパイル、ダイナミックリンクされた a.out および elf 実行ファイルの互換機能を得ることができます。
