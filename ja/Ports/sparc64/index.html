NetBSD/sparc64

NetBSD/sparc64 について 

NetBSD/sparc64 は Sun の 64ビット UltraSPARC 機 への NetBSD のポートです。 これは、 Sbus ベースの UltraSPARC システムの一部 (Ultra1 と Ultra2) 、 そして UltraSPARC IIe ベースのシステム (UltraAXi) はもちろん UltraSPARC IIi CPU (Ultra5, Ultra10 と SPARCengine AXi) ベースで PCI ベース の UltraSPARC システムの多くをサポートしています。

また、 UltraSPARC I & II PCI システムの一部のサポートもしており、 ユーザーからは Ultra 30、 60、 250 および 450 で -current を使うことに成功したとの報告があります。

NetBSD/sparc ポートのほかのドライバーは 一度移植されれば動くようになるでしょう。

現在、 NetBSD/sparc64 は一定の Ultra プラットホームをサポートしています (下を見てください)。 32ビットカーネルはほとんどの NetBSD/sparc バイナリーを実行します。 (カーネル精査ツールは例外で、修正される事を期待します)。 64ビットカーネルでは 64ビットバイナリーが走り、 そのうえ、ほとんどの 32ビットバイナリーを走らせるサポートも行われています。 (だけど、これはまだ開発中です).

簡単にインストールできる NetBSD/sparc64 のバイナリー配布物は 1.5.2 リリース と、 NetBSD-current のスナップショットがあります。 1.5 での sparc64 サポートは全般的に貧弱ですので、 NetBSD-current スナップショットを使うことを強くおすすめします。

Valeriy E. Ushakov が、 再生と録音の両方に対応した、 Ultra5 および Ultra10 マシン向けの オーディオサポートを追加しました。

2001-08-23: 新しい -current スナップショット
2001-08-21 現在のソースに基づく新しい -current のスナップショットが ftp サーバーの NetBSD/sparc64 スナップショット 置き場から入手できます。

2001-08-05: non-faulting ロードのサポート
Eduardo Horvath が non-faulting ロードのサポートおよび、 VIS 命令を使う 新しい memset/memcpy ルーチンのサポートを追加しました。

2001-07-11: NetBSD 1.5.1 リリース
21 のアーキテクチャーをサポートした NetBSD 1.5.1 がリリースされました。 より詳しい情報は 1.5.1 リリース声明にあります。

2001-02-26: NetBSD/sparc64 LKM が動きます
Matthew Green は NetBSD/sparc64 ポートの LKM を改修し、 <sys/lkm.h> の小さな変更を コミット しました。 これを利用するには、カーネルと modload(1) を、修正された <sys/lkm.h> を使って再構築する必要があります。 古いカーネルで LKM をロードしようとするとパニックし、 古い modload(1) を使うと ENOTTY (デバイスへの不適切な ioctl )が帰ってくるでしょう。

Update: これは netbsd-1-5 ブランチに組み込まれたので、 NetBSD 1.5.1 には入るでしょう。

2001-02-10: 新しい COMPAT_SVR4_32 オプション
Eduardo Horvath が COMPAT_SVR4_32 オプションを NetBSD-current に追加しました。 これをカーネルコンフィグレーションファイルに含めることで、 NetBSD/sparc64 で 32ビット SVR4 バイナリーの互換機能が得られます。

2001-02-04: 新しい netbsd32_compat14 と netbsd32_compat15 パッケージ
Matthew Green は netbsd32_compat14 と netbsd32_compat15 パッケージを pkgsrc に追加しました。 この2つのパッケージ には 32ビット NetBSD/sparc a.out 共有ライブラリー(netbsd32_compat14) と 32ビット elf 共有ライブラリー(netbsd32_compat15) が含まれています。 これによって、 NetBSD/sparc64 で、 NetBSD/sparc のもとでコンパイル、ダイナミックリンクされた a.out および elf 実行ファイルの互換機能を得ることができます。
