NetBSD/next68k

About NetBSD/next68k

NetBSD/next68k is a port of NetBSD to the 'black' 68k NeXT hardware.

It currently only runs on 68040 non-turbo (25 MHz) NeXT boxes. There is some code in locore that uses 68040 instructions to turn on the MMU, although it shouldn't be too hard to make the '030 work. Turbo NeXT machines (33Mhz) are not yet supported.

NetBSD/next68k News

Dec 27, 2000: NetBSD/next68k SCSI support update
Darrin B. Jewell has spent some time reviving the SCSI effort for NetBSD/next68k. At this point, he's using a SCSI mounted root partition and things seem to be working. Both diffs relative to the netbsd-1-5-RELEASE tree and a GENERIC kernel with scsi enabled are available for download. More information is available in Darrin's message in the port-next68k mail archive.

Dec 06, 2000: NetBSD 1.5 released
NetBSD 1.5 released with support for 20 architectures. More information is available in the 1.5 release announcement.
Archive of older news items

Supported computer systems
  • 25 Mhz 68040 Mono cube (NeXT_X15)
  • 25 Mhz 68040 Mono slab (NeXT_WARP9)
  • 25 Mhz 68040 Color slab (NeXT_WARP9C)

People

Darrin B. Jewell is the portmaster, who tries to spend around 1day/week working on this when he can.

Ongoing work

  • Finishing the scsi driver.
  • Matt DeBergalis is working on writing a wscons console driver.

Recently completed work

  • fixing the date routines in clock.c
  • testing out compiling natively with egcs.
  • figure out the rom interface, make reboot & halt work.
  • figure out the hardware powerdown.

Available projects

Anyone else interested is more than welcome to lend a hand - particular projects of interest include:

  • Fixing support for non-512 byte sector disks since NeXTStep disks are often formatted this way.
  • resyncing the bus_dma.c with the alpha code
  • cleaning up the current bus_space/intio kludge.
  • figuring out/fixing the egcs fpsp.o cross compile problem (This may be fixed. See [PR 3749]) and building a new cross compiler.
  • add 68030 support in locore.s
  • writing a floppy driver & support for floppy install.
  • decipher the Turbo NeXT hardware.
  • volunteer to recompile, make snapshots, and help other people get booted.
