NetBSD/newsmips

About NetBSD/newsmips

NetBSD/newsmips is the port of NetBSD to SONY's MIPS based "NET WORK STATION" series. More information on these machines is available.

NetBSD/newsmips port is based on NetBSD/{mips,pmax} and 4.4BSD Lite2.

Currently status is "Experimental ports", yet.

m68k based NEWS machines are supported by NetBSD/news68k port.

NetBSD/newsmips News

Dec 06, 2000: NetBSD 1.5 released
NetBSD 1.5 released with support for 20 architectures. More information is available in the 1.5 release announcement.

Nov 13, 2000: MI wscons support
NetBSD/newsmips has switched to MI wscons. Currently NWB-253 on NWS-3470D and "XA" framebuffer on NWS-5000 are supported.

Oct 30, 2000: Support for SCSI on NWS-5000
Support for the HP SPIFI3-SE SCSI interface used on news5000 is added.

Oct 12, 2000: Support for tulip based NWB-5852A
Atsushi Onoe has added support for the tulip based 100BaseTX card NWB-5852A, which is an APbus board with DEC-21140A chip. It supports 100BaseTX only. Half duplex/Full duplex can be specified manually, but there is no auto negotiation functionality.

Supported System Models
Currently only single CPU models are supported in NetBSD/newsmips.
  • R3000 and hyper-bus based models:
    • NWS-3470D
    • NWS-3410
    • NWS-3720
  • R4000/4400 and apbus based models:
    • NWS-5000
Supported Peripherals
  • On-board LANCE ethernet interface (le)
  • On-board SONIC ethernet interface (sn)
  • On-board SONY CXD1185 SCSI interface (sc)
  • On-board HP SPIFI3-SE SCSI interface (spifi)
  • Most SCSI devices
  • On-board Z8530 serial interface (zsc)
  • keyboard and mouse
  • NWB-253 framebuffer (text only)
  • "XA" framebuffer on news5000 (text only)
  • NWB-5852A 100BaseTX ethernet interface (tlp)
Not yet supported models/devices/functions
  • IOP based systems (NWS-3800 series)
  • R4600 based systems (NWS-4000 series; on going)
  • internal floppy disk
  • other framebuffers not listed above
  • X server
  • COMPAT_NEWSOS
  • etc. etc.
