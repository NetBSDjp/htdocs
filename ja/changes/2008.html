2008 年 9 月

19 Sep 2008 - Video4Linux2 互換インターフェースと、USB ビデオキャプチャー対応 (top)

計算機科学の学生でありストップモーションフレームキャプチャープログラム Frameworks の作者でもある Patrick Mahoney が、最近、 2008 Google Summer of Codeの一環として、 デバイス独立のビデオドライバー対応と、USB ビデオデバイスドライバーのコードを書きました。

NetBSD 開発者の Jared D. McNeill が、Mahoney による video(4), video(9), uvideo(4) ドライバーをインポートしました。 また、McNeill は Sony PLAYSTATION(R) Eye USB webcam 用の pseye(4) ドライバーのコードを書いてインポートしました。

これらは、Video4Linux2 互換のビデオキャプチャーインターフェースを実現するものです。 video レイヤーは read() をはじめとする Video4Linux2 API 群を実装しており、 audio(9) インターフェースのようなデバイス独立の方法での制御用 ioctl 群があります。 uvideo ドライバーは、任意の webcam 用というわけではなく、 USB ビデオクラスのデバイス専用です。 (USB ビデオクラス仕様へのリンク)

McNeill によれば、mplayer, vlc, ffmpeg, cheese (apple の Photo Booth クローン), ucview (EeePC 上の簡単なキャプチャーアプリケーション), gstreamer (GNOME アプリケーションが使っているメディアの枠組み), ekiga (SIP/IAX/NetMeeting 互換のビデオ会議ツール) など、多くの Video4Linux2 アプリケーションが動作するとのことです。 (McNeill は、pkgsrc/net/ekiga-devel パッケージもインポートしています。)

31 Jul 2008 - FFS へのメタデータジャーナリングに対応 (top)

Simon Burge が、FFS (fast file system) へのメタデータジャーナル対応コードを NetBSD-current に追加しました。

このジャーナリングコード (WAPBL - Write Ahead Physical Block Logging と呼ばれます) は、もとは Darrin B. Jewell が Wasabi Systems, Inc. のために書いたもので、 Wasabi から NetBSD コミュニティーに今年の早いうちに寄与されました。 Wasabi では WAPBL を使った製品を 2003 年から販売しています。

Simon Burge, Greg Oster, Antti Kantee, Andrew Doran は、かなりの時間を費やし、 このコードを NetBSD-current に統合し、 既存ファイルシステムを簡単に変換するために、 ジャーナルログ領域を既存ファイルシステムに自動作成できるよう拡張しました。

彼らはまた、性能ベンチマークをいくつかおこないましたが、 WAPBL の性能は soft-dependencies 有効でのマウントに非常に近いものであり、 (安全ではない) async でのマウントより少しだけ遅いものでした。 クラッシュや予期せぬ電源断において、 ジャーナルされたファイルシステムではブート時に長時間のファイルシステムチェックは必要なくなり、 カーネルが数秒の間にログを再生することになります。 これにより、クラッシュの高速な復旧が可能となり、 総停止時間の現象と高い可用性が実現できます。

既存のシステムをこのログ機能を使うよう変換するのは、 (カーネルとユーザーランドの) 更新と同じぐらい簡単です。 カーネルオプション WAPBL を選択し (すでに GENERIC カーネルの標準状態はそうなっています)、 /etc/fstab に “log” オプションを追加してから再起動します。 なお、WAPBL には soft-dependencies との互換性はありませんので、 “softdep” オプションを使っている場合は必ず削除してください。 さらなる情報は、wapbl(4) マニュアルページをご覧ください。

WAPBL は今後予定されている NetBSD 5.0 リリースで使えるようになる予定です。

このコードを寄与してくださった Wasabi Systems, Inc. と、 このコードの統合を手伝ってくださったすべての皆さん、特に ファイルシステム内ジャーナルの作業をした Simon Burge に、大いに感謝します。

30 Jul 2008 - NetBSD が永続的な慈善団体の資格を獲得 (top)

The NetBSD Foundation が、米国法の下で、 永続的な 501(c)(3) 慈善団体の資格を認められました。TNF は 2004 年から 501(c)(3) 慈善団体となっていますが、 これまでは期限つきの扱いでした。 今回認められた永続的な資格は、 170(b)(1)(A)(vi) とも呼ばれるものです。

この永続的な「公益慈善団体」の資格を得るために、 私たちが一般からの寄附によっていることを証明するため、 過去 5 年間の財務活動をもとに、必要な書類への記録をおこないました。 Lex Wennmacher, Martin Husemann, Christos Zoulas は、何回もの週末を費やして、 すべての数字を調べあげ、書類を完成させました。 この書類を IRS が受理したと報告できることをうれしく思います。

私たちが公益慈善団体であることは、課税条件が最も有利となるという利点にくわえ、 雇用者のマッチングドネーションを受けやすくなるという点で、 重要なことです。

NetBSD プロジェクトは、Google の “Summer of Code” プログラムに、4 年連続で指導組織として参加できることを誇らしく思います。 また、今年の採択プロジェクト一覧を発表できることを喜ばしく思います。 14 の提案が採択され、これらは幅広い分野にわたっているので (“すべての人のもの!”)、 この夏が面白いものになることを楽しみにしています。

プレスリリースに、 各プロジェクトの詳細を含めた一覧を掲載しています。

02 Apr 2008 - Summer of Code 学生応募期限の延期 (top)

Google が、今年の Summer of Code の学生応募期限を 2008 年 4 月 7 日までに延期しました。 今から提案を提出しても、まだ間に合うということです。 この延期によって、学生の皆さんは、指導者とやりとりしたり、 メーリングリストで提案について議論したり、 提案を洗練して提出するための意見を求めたりするための時間を、 もっと取れることになりました。

NetBSD プロジェクトでは、プロジェクト一覧のページに載っていない独自の提案も、 一覧にある提案と等しく歓迎していますが、 このことを今回の期限延期を機に皆さんに思い出していただきたいとも考えています。 NetBSD をよりよくする方法について、よい考えをお持ちの方は、 その考えをふくらませて応募してください。

Summer of Code プログラムの採択は、厳しい競争となるので、 提案をもうひとつ提出したいという方もいらっしゃるでしょう。 複数の方が同じ取り組みを応募してくることがよくあるので、 そのなかから一人の学生を選ぶのに苦労しています -- 質のいい応募がもうひとつあれば、あなたが選ばれる可能性が増えるかもしれません。

何か質問があれば、適切なメーリングリストにメールを出すか、 IRC (freenode.net) の #netbsd-code で、 指導者になってくれそうな人を探してみるなどしてください。

20 Mar 2008 - 祝・NetBSD 15 歳の誕生日 (top)

今週は、現に保守されている自由に利用可能なオペレーティングシステムとして最も古い部類である NetBSD オペレーティングシステムの 15 周年にあたります。 NetBSD のソースコードのリポジトリーへの最初の commit は、 1993 年 3 月 21 日でした。NetBSD の 15 周年祝いについて詳しくは、 このお知らせをご覧ください。 イベント展示室に 15 周年記念宴会の一覧を掲載する予定です。

17 Mar 2008 - Google Summer of Code 2008 に NetBSD が再び参加 (top)

Google が、 Summer of Code プロジェクトを今年もおこなっています。 これは、学生の皆さんに、夏休みの間、アルバイトするかわりに、 オープンソースプロジェクトのコード書きを楽しんでもらうという趣旨のものです。 NetBSD プロジェクトは、今回で 4 回目になりますが、 Summer of Code プロジェクトの指導組織に選ばれました。

NetBSD のプロジェクトに取り組むことに関心のある有望な学生の皆さんは、 取り組みたいプロジェクトについて検討を始め、 NetBSD の公開メーリングリストで議論をしていただくとよいでしょう。 私たちは、参考になるよう、 GSoC プロジェクトの提案一覧と、 (より困難な) 一般的なプロジェクト一覧を用意しています。 また、Google に最終的なプロジェクト提案を送る際に書いておくべき事項についての NetBSD Project Application/Proposal HowTo もご覧ください。 この情報は、提出されたプロジェクト提案を私たちが評価するために役立ちますが、 あなたのプロジェクトを有利にするためにも役立つのです。

28 Feb 2008 - 2007 年 NetBSD 募金キャンペーンの結果 (top)

NetBSD はオープンソースオペレーティングシステムのプロジェクトであり、その運営は資金提供に依存しています。Google からの寛大な寄附のおかげで、私たちは対称型マルチプロセッシング (SMP) の分野での開発に資金をまわすことができるようになりました。そして、この開発への資金提供をさらに続けるために、募金活動を開始しました。企業・利用者・コミュニティーの皆さんから多くの支援をいただいたおかげで、2007 年の募金キャンペーンの目標だった 5 万米ドルを達成することができました。 支援してくださった企業が NetBSD についてどう述べているか、 最近どのような成果があがっているか、 また、寄附によってどんな変化が期待できるかなど、 さらなる情報は、 2007 年 NetBSD 募金キャンペーンの結果に関するプレスリリースをご覧ください。

13 Feb 2008 - ポートごとの変更・更新 (top)

NetBSD/ofppc

  • Tim Rightnour が ofppc ポートの復活を宣言しました: "このポートは基礎から書き直され、 本質的に NetBSD の新ポートといえるものになりました。" (発表)

  • Tim が NetBSD/ofppc の新ポートマスターに就任しました: "NetBSD の PowerPC への最初の移植において優れたはたらきをした、 前ポートマスターの Wolfgang Solfrank に感謝したいと思います。" (発表)

  • IBM 7044-270 (POWER3-II cpu) マシンに対応しました。 (発表)

NetBSD/prep

最近、Tim Rightnour が NetBSD の powerpc ポートを作り直しました。 Kevin Bowling の支援を受け、彼は NetBSD を IBM MCA RS/6000 model 7006 に移植しました。これにより、 NetBSD はこのクラスのマシンで動作する初のフリーオペレーティングシステムとなりました。 (発表)

NetBSD/xen

Manuel Bouyer が、 PAE 拡張を使った Xen/i386 DomU への対応と、Xen/amd64 (Dom0 および DomU) への完全対応を NetBSD-current に実装しました。 NetBSD/i386 には汎用的な PAE 対応がまだありませんが、 NetBSD i386-PAE DomU は最近の Linux x86_32 および x86_64 Xen ハイパーバイザー上で、そのままで動くようになりました。 (発表: 1, 2)

NetBSD/i386

Hubert Feyrer が、基本的なコンパイル済バイナリーパッケージの附属した NetBSD/i386 用インストール CD を発表しました。 (詳細)

NetBSD/pc532 終了

NetBSD の PC532 プラットフォームへの対応は、過去の話になりました: Simon Burge が、このポートの終了を発表しました。 (発表)

NetBSD/arm

Donald T. Hayford が、 NSLU2 上での NetBSD の構築およびインストールの説明を投稿しました。 (説明: 1, 2)

NetBSD/hp700

Nick Hudson が、HPPA アーキテクチャーへの NetBSD ポート (NetBSD/hp700) の最新の進捗報告を投稿しました。 (状況報告)

24 Jan 2008 - Joachim Schueth: Colossus 暗号解読機を打ち負かす (top)

Joachim Schueth は、2007 年 11 月の暗号解読コンテストで、有名な Colossus Mark II 暗号解読機を打ち負かしました。 Colossus コンピューターは、第二次世界大戦において、 ドイツの暗号メッセージの解読に使われました。 Schueth は、NetBSD で動くラップトップ、暗号に関する深い知識と、Ada プログラミング言語を使って、 このコンテストに勝利しました。私たちは、 この暗号イベントや彼が使った道具の歴史的および技術的な背景について、 彼の話を聞きました - インタビューを読む
