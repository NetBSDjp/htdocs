Formal NetBSD Releases

Formal releases of NetBSD are intended to provide a stable, supported platform on which people can base their work. They provide a balance between features and stability. They are also typically easier to install than NetBSD-current.

A release is done by creating a branch of the source tree. After a first test of the release building tools has been done (e.g. NetBSD-1.5_ALPHA) it is frozen; that is, no more features may be added, and bug fixes may only be added by -- or on consent of -- the release engineer. Usually one or more beta release binaries (e.g. NetBSD-1.5_BETA) are built per architecture for testing purposes, before the final release (e.g. NetBSD-1.5).

During this time, many developers concentrate on the release branch. This is why after a release, the changes on the release branch are merged back into the main branch.

Why use a formal release?

Because they are well-tested and made relatively infrequently, formal releases are useful for people who don't want to be on the "bleeding edge" of development, or who just want to run applications. Since formal releases are effectively unchanging (there are typically very few official patches made to them), they are relatively easy to support, both by NetBSD's developers and the developers of applications. The biggest problem with running the latest formal release of NetBSD is that you don't have access to the latest features and bug fixes in the development tree.

Formal releases are relatively easy to install. All of the platforms supported by a formal release come with a detailed set of installation instructions, and most also include some sort of bootstrapping media. (For example, the i386 port typically uses floppy disk images to bootstrap, while the amiga and hp300 ports use a miniroot filesystem image.) There is also typically an upgrade procedure to bring a system from the previous formal release to the current one.

In short, if you're not the type of person who'd be willing to upgrade your operating system on a semi-daily basis, or if you want a system that is relatively easy to install and upgrade, a formal release of NetBSD is for you.

How to obtain updates for formal releases

For those who do not wish to wait for the next release, we have daily updated snapshots of the release branches available via both FTP and SUP.

The directories /pub/NetBSD/NetBSD-release-1-4 and /pub/NetBSD/NetBSD-release-1-5 contain the extracted sources plus weekly updated tar files of both the 1.4 and 1.5 release branches respectively. These files are created in a similar manner to those in the /pub/NetBSD/NetBSD-current directory.

A history of formal releases

Thus far, there have been 17 formal releases of NetBSD. Eight releases were major releases, which introduce new features in the operating system; and nine were patch releases, which fix bugs in a major release and improve its stablility.

The current release

  • NetBSD 1.5, released on December 6, 2000
    • NetBSD 1.5.2 (patch release) released on September 13, 2001
    • NetBSD 1.5.1 (patch release) released on July 11, 2001

Previous releases

  • NetBSD 1.4, released on May 12, 1999
    • NetBSD 1.4.3 (patch release) released on November 25, 2000
    • NetBSD 1.4.2 (patch release) released on March 19, 2000
    • NetBSD 1.4.1 (patch release) released on August 26, 1999
  • NetBSD 1.3, released on January 4, 1998
    • NetBSD 1.3.3 (patch release) released on December 23, 1998
    • NetBSD 1.3.2 (patch release) released on May 29, 1998
    • NetBSD 1.3.1 (patch release) released on March 9, 1998
  • NetBSD 1.2, released on October 4, 1996
    • NetBSD 1.2.1 (patch release) released on May 20, 1997
  • NetBSD 1.1, released on November 26, 1995
  • NetBSD 1.0, released on October 26, 1994
  • NetBSD 0.9, released on August 23, 1993
  • NetBSD 0.8, released on April 20, 1993
