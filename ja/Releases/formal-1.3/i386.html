NetBSD/i386 1.3

NetBSD/i386 1.3 は、 NetBSD/i386 の 6 度目の公式リリースです。
NetBSD 1.3.3 はNetBSD 1.3より優先して使うべき、パッチリリースです。

Charles Hannum と Frank van der Linden が NetBSD/i386 のメンテナーです。

対応ハードウェア

NetBSD/i386 1.3 は、i386ファミリープロセッサーを搭載した ISA(ATバス)、EISA、 PCI、VLバスシステムで、数値演算コプロセッサーの有無に関わらず、動作します。 IBM PS/2 のような MCA システムには対応して*いません*。 最小の構成では 4MB の RAM と 50M のディスク容量が必要とされていますが、 今日このような最小構成のシステムを動かしているような人を私たちは知りません。 完全なシステムのインストールにはより多くのディスクスペースが必要ですし (ソース以外のバイナリー配布を展開するには、スワップスペースその他を除いても、 最低 65M は必要です) 、 X の実行やシステムのコンパイルにはより多くの RAM を推奨します。 (RAM が 4MB でも確かに X の実行やコンパイルは可能ですが、速くはありません。 16 MB ほどの RAM もない場合は、 CPU を速くするより RAM を増やすほうが 重要なのです。)

"[b]" 印の付いたデバイスのサポートには、 BIOS がマザーボードの PCI-PCI ブリッジをサポートしていることが必要です。最近の安価な Pentium マザーボード のほとんどは、これをサポートしているか、または BIOS アップグレードで対応可能です。

今のところ*未*対応だが、よく質問されるハードウェア:

  • AMD PCscsi SCSI ホストアダプター (PCnet-SCSI の PCnet 部はしっかり動作しますが)
  • マルチプロセッサー Pentium および Pentium Pro システム。 (プロセッサー 1 個だけを使ってならしっかり動くはずですが。)
  • NCR 5380 ベースの SCSI ホストアダプター。
  • PCI WD-7000 SCSI ホストアダプター。
  • QIC-40 および QIC-80 テープドライブ。 (これらは、 フロッピーディスクコントローラーに接続するテープドライブです。)
将来的にはこれらのデバイスの多くにも対応したいと考えています。

もしNetBSD/i386 1.3のインストールに興味があれば、 インストールノートを見るべきです。 X のインストールで問題があった場合は、 XFree86 Documentation を確認すべきです。

i386に特化した port-i386を含めた、 さまざまなNetBSDの メーリングリストがあります。
NetBSD/i386の入手方法: CD-ROM、 あなたに近いftpミラー、あるいは ftp.NetBSD.org から直接入手。
