NetBSD リリース用語集と図

NetBSD は 3つの形態で提供されます:

  • 公式リリース - テストされた製品バージョン.
  • NetBSD-release - 保守用の (パッチ) ツリー.
  • NetBSD-current - 主要な (開発) ツリー.

また例として NetBSD-1.3 の歴史と リリース枝の概観 を表す図を示します。

NetBSD-current
これは NetBSD 開発の "流血の刃先" です。このバージョンは 常に数字に続いて文字がつく形式になっており、次の 公式リリースへと開発されていきます。(つまり、1.3Fは 最終的に 1.4 となります)。 文字はカーネル/ユーザーランド-インタフェースが変更されると ひとつ進みます。

これは SUP, CVS, cvsweb や ftp://ftp.netbsd.org/pub/NetBSD/NetBSD-current からの ftp で利用できます。

BSD CSRG 用語で気づくと思いますが、-current はふつうアルファ品質での配布です。 これはコンパイルできることさえ保証されていません。

公式リリース
公式な、信頼できる NetBSD のソースとバイナリーの配布です。

リリース後の重要なバグ修正はリリース枝に 反映されて、パッチリリースされます(例えば NetBSD-1.4.1)。 パッチリリースは同じ日付の -current よりも信頼性はあるが、 ない機能があることにも気づくと思います。 -current とパッチリリースを混ぜる作業は特に行なわれません。

NetBSD-release
これは日々更新されるリリース枝のスナップショットで、 FTP と SUP により利用できます。

リリース枝は最近のリリースと、次のパッチリリースに 含まれるバグ修正からなります。つまり、最近のリリースが 1.5ならば、NetBSD-release-1-5 は 1.5.1の 前の状態と考えることができます。

これは、最近のリリースの最新の修正がほどこされた状態を手に入れる 簡単な方法として考えることができます。

/pub/NetBSD/NetBSD-release-1-4 と /pub/NetBSD/NetBSD-release-1-5 ディレクトリーに、 それぞれ 1.4 と 1.5 リリース枝の 展開されたソースと毎週更新される tar ファイルが置かれています。 これらのファイルは /pub/NetBSD/NetBSD-current ディレクトリーと同様な方法で 作成されています。

Snapshots
スナップショットは通常 port メインテイナーにより 作成され、NetBSD-current と NetBSD-release のどちらでも あり得ます。通常、-current を追跡しはじめるのにもっとも 簡単な方法は最新のスナップショットをインストールする事でしょう。
