NetBSD release glossary and graphs

NetBSD is available in three forms:
  • formal release - tested, production version.
  • NetBSD-release - maintenance (patch) tree.
  • NetBSD-current - feature (development) tree.
Additionally, there are graphs showing the history of NetBSD-1.3 and an overview of release branches as an example.

NetBSD-current
This is the "bleeding edge" of NetBSD development. The version is always in the form of a number followed by a letter, and will develop into the next formal release. (eg: 1.3F eventually became 1.4). The letter is incremented when the kernel/userland-interface is changed.

It is available via SUP, CVS, cvsweb, and ftp in ftp://ftp.netbsd.org/pub/NetBSD/NetBSD-current.

You should be aware that in BSD CSRG terms, -current is normally an alpha quality distribution. It isn't even guaranteed to compile.

formal release
An official, believed to be reliable source and binary distribution of NetBSD.

After a release is done, urgent bug fixes are ported back to the release branch, and patch releases (e.g. NetBSD-1.2.1) are done. You should be aware that a patch release is more reliable than -current of the same date, but is missing features. Most particularly, it usually doesn't work to mix a patch release with -current.

NetBSD-release
This is a daily updated snapshot of the release branch, available via both FTP and SUP.

The NetBSD-release branch is the last release, plus whatever bug fixes will be going into the next patch release. So if the latest release is 1.4.1, NetBSD-release can be thought of pre-release 1.4.2.

It can be considered an easy way to get the most up to date fixes for the most recent release.

The directory ftp.netbsd.org/pub/NetBSD/NetBSD-release contains the extracted sources plus weekly updated tar files. These files are created in a similar manner to those in the /pub/NetBSD/NetBSD-current directory.

snapshots
Snapshots are usually done by a port maintainer, and can be of NetBSD-current or NetBSD-release. Usually the easiest way to start tracking -current is to start by installing a recent snapshot.
