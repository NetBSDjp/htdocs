NetBSD release glossary and graphs

NetBSD is available in three forms:

  • Formal release - tested, production version.
  • NetBSD-release - maintenance (patch) tree.
  • NetBSD-current - feature (development) tree.

Additionally, there are graphs showing the history of NetBSD-1.3 and an overview of release branches as an example.

NetBSD-current
NetBSD-current is the main development branch of NetBSD, it's the "bleeding edge" of NetBSD development. The version is always in the form of a number followed by a letter, and will develop into the next formal major release. Eg: 1.5F eventually became 1.6. The kernel's version letter is incremented when the interface between kernel and userland is changed.

NetBSD-current is available via SUP, CVS, cvsweb, and FTP in ftp://ftp.netbsd.org/pub/NetBSD/NetBSD-current.

You should be aware that in BSD CSRG terms, -current is normally an alpha quality distribution. It isn't even guaranteed to compile.

Formal release
An official, believed to be reliable source and binary distribution of NetBSD. Consists either of a major release like 1.5, or of a patch release like 1.5.1.

After a release is done, urgent bug fixes are ported back to the release branch, and patch releases (e.g. NetBSD-1.4.1) are done. You should be aware that a patch release is more reliable than -current of the same date, but is missing features. Most particularly, it usually doesn't work to mix a patch release with -current for both userland and kernel.

NetBSD-release
A release branch is the last release, plus whatever bug fixes will be going into the next patch release, pulled up from the NetBSD-current development branch. E.g. if the latest release is 1.5, the release branch for it is "netbsd-1-5" which can be thought of containing an alpha-version of 1.5.1.

The NetBSD-release branch(es) can be considered an easy way to get the most up to date fixes for the most recent release.

There are daily updated snapshots of the release branches, available via both CVS, FTP and SUP. The directories /pub/NetBSD/NetBSD-release-1-4 and /pub/NetBSD/NetBSD-release-1-5 contain the extracted sources plus weekly updated tar files of both the 1.4 and 1.5 release branches respectively. These files are created in a similar manner to those in the /pub/NetBSD/NetBSD-current directory.

Snapshots
Snapshots are usually done by a port maintainer, and can be of NetBSD-current or NetBSD-release branches. Usually the easiest way to start tracking -current is by installing a recent snapshot.
