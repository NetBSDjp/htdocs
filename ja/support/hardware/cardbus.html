NetBSDでサポートされている PCMCIA ハードウェア

機種に依存しない PCMCIA CARDBUS ドライバー

下記のデバイス(括弧内はドライバー名)が機種に依存しない PCMCIA CARDBUS ドライバーによってサポートされています。 リストにないデバイスは、各プラットフォームのページをチェックしてみてください。 そのデバイスのドライバーが、まだ機種独立になっていないのかもしれません。

別の可能性として、そのデバイスはリストにあるデバイスと互換かもしれませんし、 そのデバイスを見落していて、リストに載せ忘れているのかもしれません。 不明な点がある時は、 -current のカーネルを起動するか、 <netbsd-users@NetBSD.org> にメールしてください。

訳註

netbsd-users は英語のメーリングリストです。日本語での問い合わせは、 ここを参照してください。

このページはNetBSD-currentの現状を反映していることに注意してください。 特定のリリースでサポートされているハードウェアを調べるには、 そのリリースのインストレーション・ノートを参照してください。

サポートされている PCMCIA Cardbus デバイス

Hayakawa Koichi は、彼の作った Cardbus コードを -current に統合しています。 これによって、さまざまな Yentaベースの Cardbus ブリッジと関連する デバイスドライバーがサポートされます。

今のところ、下記の Cardbus デバイスのデバイスドライバーが存在しています。

  • SCSI ホストアダプター
    • Adaptec AIC-7860 (ahc(4)).
    • AdvanSys ASC1200[AB] および ABP-9xxU[A] (adv(4)).
    • I-O Data および WorkBiT NinjaSCSI (njs(4)).
  • ネットワークインターフェース
    • 3Com 3c575 10/100 イーサネットカード (ex(4))
    • Intel i82557 ベースの 10/100 イーサネットカード (fxp(4))
    • RealTek RTL8139 ベースの 10/100 イーサネットカード (rtk(4))
    • Digital 21143 ベースの 10/100 イーサネットカード (tlp(4))
    • Xircom X3201-3 ベースの 10/100 イーサネットカード (tlp(4))
    • ADMtek AN983 と AN985 ベースの 10/100 イーサネットカード、 Planex, Corega, I/O Data, AboCom, ACCTON カードを含みます (tlp(4))
    • Atheros 802.11 ワイヤレスネットワークカード (ath(4))
    • ADMtek ADM8211 802.11 ワイヤレスネットワークカード (atw(4))
    • Ralink Technology RT25x0 802.11a/b/g ワイヤレスネットワークカード (ral(4))
    • Realtek RTL8180L 802.11b ワイヤレスネットワークカード (rtw(4))
  • コミュニケーションコントローラー
    • 16550 と その互換の UART、モデムカード (com(4))
    • USB Open Host Controller (ohci(4))
    • USB Enhanced Host Controller (ehci(4))
    • IEEE 1394 (Firewire) Open Host Controller (ehci(4))
