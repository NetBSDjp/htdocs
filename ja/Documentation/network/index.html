始めよう

NetBSD のインストール手順は基本的なネットワークの設定を含んでおり、 標準的なワークステーションの設定を実装しています。 この設定は NetBSD の多くのネットワークの機能を利用するために拡張することができます。

  • 参考:

    • RFC 1918 - Address Allocation for Private Internets

ネットワーク設定ファイル

ネットワークの設定は、テキスト形式の設定ファイルによって 設定します。
  • /etc/rc.conf - システムの初期化時に、ネットワークサービスを含む システムサービスを自動的に開始するための設定。
  • /etc/hosts - もっとも基本のホスト名 /IP マップ。
  • /etc/myname - FQDN (ドメイン名のついたホスト名)
  • /etc/ifconfig.{IF}. - システムの初期化時、ネットワークインタフェースを 設定するために /etc/netstartで用いるネットワークインタフェース IF を定義。
  • /etc/ifaliases - 一つのインタフェースには一つ以上の IP アドレスを割り当てられる。 ここで記述された alias は、システムの初期化時に /etc/netstart で用いられる。

ネットワーク設定プログラム

ネットワークに新しいホストを追加する

ネットワークにホストを追加するのに最低限、 ローカルのネットワークのアドレスを設定したインタフェースが必要です。 NetBSD のインストール中にネットワークの情報が提供されると、 新しいホストはすぐにネットワークに接続され、 その IP アドレスでアクセスされます。

ネットワークの新しいホストを名づける

IP アドレスと同じようにホスト名でのアクセスを可能にするには、 新しいホスト名とその IP アドレスをネットワーク設定ファイルに 追加します。それにはいくつかの方法があります;
  1. /etc/hosts: いくつかのホストからなる小さなネットワークでは、 ホスト名と IP のマップを各ホストの /etc/hosts ファイルに 手でコピーする

    /etc/hosts:
    192.168.1.2 host2.mydomain.org.au host2

  2. yp はネットワーク上の /etc/hosts ファイルが自動的に同期 がとれるようにする (他の設定ファイルによる)。
  3. DNS: ホスト名のマップをネームサーバ named からアクセスされるゾーンファイルにまとめられるようにする。 (DNS ファイルの設定や管理についての多くのドキュメントが www.dns.net にあります)。

    正引きゾーンファイルのエントリ
    host2 A 192.168.1.2

    逆引きゾーンファイルのエントリ
    2 PTR host2.mydomain.org.au.

インターネットワーキング

あなたのネットワーク (一つのホストから成るかもしれない) を 他のネットワークにつなぐ場合、少なくとも一つのホストを 二つのネットワーク間のゲートウェイとして動作させなくてはなりません。 ゲートウェイホストは二つのネットワークインタフェースを持っており、 それぞれを各ネットワークについて設定します。
  • 二つ目のイーサネットのインタフェース設定ファイルを作成する (/etc/ifconfig.{interface}) 。

    /etc/ifconfig.we1
    192.168.2.30 netmask 0xffffff00 media 10base2/BNC

  • 新しいインタフェースを有効にする為にリブートする。

PPPによるインターネットワーキング

シリアルポート (モデムの有無にかかわらず) 上にネットワークインタフェースを pppd を用いて作成できます。

PPP のセットアップには多くの方法があります。あなたの ISP に接続するのに 適した一つの簡単な方法は;

  1. ピアオプションファイルを作成する

    /etc/ppp/peers/myisp
    # Example pppd options
    # Specific for myisp
    /dev/tty01
    local_IP_address:remote_IP_address
    connect '/usr/sbin/chat -v -f /etc/ppp/peers/myisp.chat'
    defaultroute
    persist
    ipparam myisp
    asyncmap 0

  2. デバイスオプションファイルを接続する

    /etc/ppp/options.{ttyname}
    # Example pppd options
    # Specific for ttyname
    lock
    crtscts
    57600
    modem

  3. チャットファイルを作成する

    /etc/ppp/peers/myisp.chat
    # Example chat file
    # Specific for myisp
    ABORT BUSY ABORT 'NO CARRIER' ""
    \da\ptz0 OK
    \da\ptdt8887776655 CONNECT

  4. コネクションを確立する;

    pppd call myisp

  5. 変数をセットする

    ppp_peers="myisp" を /etc/rc.conf に書いておくとブート時に自動的にコネクションが確立する。

イーサネットインタフェースと異なり、/etc/ifconfig.{interface} ファイルを PPP インタフェースに作成する必要はありません。

    • netstat - ネットワーク状態の表示
    • systat netstat - ネットワーク接続を動的に表示

Appletalk を使う

NetBSD は unix マシンと appletalk マシンをイーサネット (localtalk では ありません) で通信を可能にする 'netatalk' をサポートしています。 これは MacOS マシンが NetBSD マシンを介してファイルシステムを読んだり プリンタを用いたり、NetBSD マシンが appletalk プリンタに印刷したり することができます。 必要なソースは ftp.netbsd.org (またはミラーサイト) の /pub/NetBSD/misc/netatalk から利用可能です。
