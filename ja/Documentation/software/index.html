NetBSD Documentation:

Installing/compiling other software

If you are running NetBSD 1.3 or later you we strongly recommend you make use of the many software packages available in the NetBSD packages collection.

Frequently Asked Questions

  • Where to find precompiled programs
  • How to build/download programs
  • Compiling fails with 'virtual memory exhausted'

Where to find precompiled programs

NetBSD 1.3 and later support a 'package' collection for many items of third party software. Precompiled binaries are available at ftp://ftp.netbsd.org/pub/NetBSD/packages, or a mirror thereof (each NetBSD port has its own subdirectory). An HTML index is also available.

To install packages you can either:

  • Copy a package file to your machine and run "pkg_add package.tgz"
  • Install direct from the ftp site. For example, to install pine on a NetBSD/sparc 1.3 system: "pkg_add ftp://ftp.netbsd.org/pub/NetBSD/packages/1.3/sparc/All/pine-3.96.tgz".

How to build/download programs

NetBSD 1.3 and later support a 'package' collection that enables freely available third-party software to be built easily on NetBSD hosts. Once the software has been built, it is manipulated with the pkg_* tools so that installation and de-installation, printing of an inventory of all installed packages and retrieval of one-line comments or more verbose descriptions are simple.

Precompiled binaries for various architectures are also available.

Compiling fails with 'virtual memory exhausted'

Try "unlimit datasize".

NetBSD sets resource limits on a per process basis. The current values can be displayed with "limit", and set with "limit resource value", or unlimited with "unlimit resource". More details are available on the getrlimit(2) manual page ("man getrlimit").
