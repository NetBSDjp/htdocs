NetBSD ドキュメンテーション:

他のソフトウェアのインストールとコンパイル

もしあなたが NetBSD 1.3 以降を動かしていたら、 NetBSD packages collection の多くのソフトウェア パッケージを利用することを 強く お勧めします。

よくある質問と回答

  • どこでコンパイル済みのプログラムを見つけられますか
  • どうやってプログラムをダウンロードやビルドしたらよいですか
  • コンパイルが 'virtual memory exhausted' と出て失敗します

どこでコンパイル済みのプログラムを見つけられますか

NetBSD 1.3 以降では、多くのサードパーティー製のソフトウェアの 'パッケージ' をサポートしています。 コンパイル済みのバイナリは、 ftp://ftp.netbsd.org/pub/NetBSD/packages やミラーサイトから利用可能です (NetBSD の各 port はそのサブディレクトリにあります)。 HTML index も利用可能です。

どちらの方法でもパッケージをインストールできます:

  • パッケージファイルをあなたのマシンにコピーして "pkg_add package.tgz" を実行する。
  • ftp サイトから直接インストールする。たとえば、NetBSD/sparc 1.3 に pine をインストールするなら: "pkg_add ftp://ftp.netbsd.org/pub/NetBSD/packages/1.3/sparc/All/pine-3.96.tgz"

どうやってプログラムをダウンロードやビルドしたらよいですか

NetBSD 1.3 以降では、NetBSD 上で簡単にサードパーティ製ソフトウェアを ビルドできる 'package' コレクションを サポートしています。一旦ソフトウェアがビルドされれば、あとは pkg_* ツールで操作することができます。このため、 インストールしたりアンインストールしたり、全てのインストール済みのパッケージの 一覧を表示したり、一行コメントや あるいはもっと詳細な説明を得たりする のは簡単です。

コンパイル済みのバイナリ がさまざまなアーキテクチャ上で利用可能です。

コンパイルが 'virtual memory exhausted' と出て失敗します

"unlimit datasize" としてみてください。

NetBSD はプロセス毎にリソースの制限を設定します。現在の値は、 "limit" で表示、 "limit resource value" で設定、 "unlimit resource" で制限を外すことができます。 詳しくは getrlimit(2) マニュアル ("man getrlimit") を参照してください。
