NetBSD packages コレクション

手短に言えば、NetBSD packages コレクションは、 NetBSDに移植されたソフトウェアユーティリティとライブラリの集まりです。

packages コレクションソフトウェアはソフトウェアを、 いくつかの手段によりインターネットを通して、 そのホームサイトから捜して持ってきて、それが正しいものか確かめ、 あらゆるパッチを当てて、NetBSDのためにそのソフトウェアを設定し、 そしてそれを構築することを行います。 あらゆる不可欠なソフトウェアは、 あなたのために同じように構築されインストールされるでしょう。 ソフトウェアのインストールとアンインストールは、 packaging ユーティリティにより管理されます。

ソフトウェアソースやバイナリをそれらのホームサイトから自分自身で得るよりも、 packagesが進んでいることをいくつか挙げます。

  • packagesはすでに、 あなたのシステムで正しくコンパイル、 インストールされるように設定されているので、 自分自身でソフトウェアを移植することを心配する必要がありません。
  • 分類された、 NetBSDで動くソフトウェアのプログラムの最新バージョンと、 そのパッチを得ることができます。
  • いったん使い方がわかれば、ダイアルアップ接続でさえ、 簡単にすばやく使うことができます。
  • 他の人に役立つように、 あなたの移植作業をpackaging システムに追加して貢献することができます。

packagesでは、コンパイル済みのbinary packagesか、 ソフトウェアをインストールおよび維持するための'pkgsrc'を使用することができます。

コンパイル済み Binary Packages のインストール

NetBSD 1.3用、もしくはより新しいNetBSD用 コンパイル済みbinary packages セレクションは、 NetBSD ftp サイトの /pub/NetBSD/packages/ にて利用できます。

おそらく大多数のユーザーは、もっとも有益な browsable web listingを見つけるでしょう。

これらのbinary packagesは、手元のディスクにダウンロードして 

    pkg_add <package>
というように pkg_add でインストールすることができますし、 
    pkg_add ftp://ftp.netbsd/org/pub/NetBSD/packages/<os_release>/<arch>/<type>/<package>
というように直接 pkg_add に完全なURLを指定して、インストールすることもできます。

packagesは、通常 /usr/pkg にインストールされます。 パス変数(PATH)に確実に /usr/pkg/bin と /usr/pkg/sbin を加えてください(一番の方法は、/etc/csh.cshrcで設定することです)。

NetBSD Packages コレクションの設定

  1. NetBSD Packages コレクションは毎週アーカイブされ、 tar+gzipされたファイルでNetBSD ftp サイトと多くのミラーサイトに置かれます。 この'package ソースファイル (pkgsrc)'は、 現在のPackages コレクション中の、 各々のソフトウェアの説明と情報が含まれています。 このpkgsrcを得てシステムにインストールすることにより、 そのコレクションに含まれるソフトウェアを簡単にコンパイル、 インストールすることができます。 まず必要なことは、 Package コレクション ソースファイル(pkgsrc)をダウンロードすることです。 NetBSDサイトから、 もしくは近くのミラーサイトから、 ftp://ftp.netbsd.org/pub/NetBSD/NetBSD-current/tar_files/pkgsrc.tar.gz を得ます。

  2. 一旦自分のシステムにpkgsrc.tar.gzをダウンロードしたら、 'su -' のようにrootでログインし、 '/usr' ディレクトリにそのファイルをコピーします。

  3. もし、以前の pkgsrc.tar.gz を展開していたならば、まずそれを削除します。 

    	rm -rf pkgsrc

  4. そしてtarで、 そのファイルを伸長し新たなホーム(/usr/pkgsrc)に展開します。 

    	tar -xzvpf pkgsrc.tar.gz

  5. コレクションからpackageを構築するときは、 通常新しいディレクトリツリー '/usr/pkg/' を作成し、 そこにインストールすることになります。 例えば、'LOCALBASE=/usr/local' を /etc/mk.conf に加えることにより、これを変更することができます。

NetBSD Packages コレクションの使用

  1. '/usr/pkgsrc/' ディレクトリには、 カテゴリ別に整理されたサブディレクトリが含まれます。 各ディレクトリでは 'README' ファイルで、 それぞれのユーティリティ/アプリケーションが何なのかを知ることができます。

  2. アプリケーションのひとつをインストールするのは非常に簡単です。 例えば、あなたのシステムに Joe というエディター(Joeは小さいが、 WordstarやEmacsのようなエディターを真似たパワフルなエディターです) をインストールするには、まず次のようにカレントディレクトリを変更します。 

    	cd /usr/pkgsrc/editors/joe

  3. もしダイアルアップ接続によりインターネットにアクセスしているならば、 ソフトウェアソースおよびあらゆるパッチを捜して持ってくるために、 この時点で接続してください。

  4. そして '/usr/pkgsrc/editors/joe' ディレクトリにて、 

    	make
    と入力します。ソースとパッチ、コンパイルに必要ないくつかのソフトウェアが あなたのシステムにダウンロードされるでしょう。

  5. ソフトウェアがダウンロードされると、パッチが適用された上で、 コンパイルされます。それにかかる時間はあなたのコンピューターによりますし、 そのソフトウェアが依存している他のpackagesの数にもよります。

  6. これが完了し、どんなエラーも表示されなければ、 ダイアルアップ接続を切断してもかまいません(電話代は節約しましょう!)。

  7. 次は新たにコンパイルされたプログラムを、 実際にあなたのシステムにインストールします。 '/usr/pkgsrc/editors/joe' ディレクトリにいる間に (もしくはインストールのためのディレクトリで) 

    	make install
    と入力してください。

  8. そのソフトウェアはまさにインストールされ、 使用できるようにセットアップされたことになります。 もうこれ以上コンパイル後の作業ファイルは必要とされないので、 

    	make clean
    と入力し作業ファイルを削除してしまってもかまいません。 もし、プログラムをコンパイルするときに、 依存関係により他のpackagesがコンパイル/インストールされたならば、 それらも次のコマンドにより、きちんと削除することができます。 
    	make clean-depends

利用することができるNetBSD Packages コレクションの見つけ方

コレクション中にどんなpackagesがあるのかを知る、もっとも良い方法は、 pkgsrc の一番上のディレクトリ(これは通常 /usr/pkgsrc でしょう)に移動し、 "make readme" と入力することです。 これにより、README.html というファイルが pkgsrc の一番上のディレクトリ、 またすべてのカテゴリとpackage ディレクトリに作成され、 lynx (pkgsrc/www/lynx) や Mozilla (pkgsrc/www/mozilla) や Communicator のようなブラウザーを使うことで、 packageの機能に付いての短い(一行)コメントと、 詳細な説明へのポインタを見ることができるようになります。 これはまた次のURLでも見ることができます。 

	ftp://ftp.netbsd.org/pub/NetBSD/packages/pkgsrc/README.html

コレクション中のpackagesを見つける別の方法としては、 pkgsrc の一番上のディレクトリに移動して、 "make index" と入力するというのもあります。 これは、"make print-index | more" で見ることができる pksrc/INDEX を作成します。 また、packagesやキーワードを "make search key=<somekeyword>" により 検索することもできます。

その他のNetBSD Packages コレクション用コマンド

NetBSD Package コレクションを使う上で、役に立つその他のコマンドは

  • make deinstall - システムにインストールされたpackageを削除します。 上記手順の逆を行います。
  • pkg_info - システムにインストールされたpackageを表示します (packagesのインストールによりデータベースに追加されています)。
