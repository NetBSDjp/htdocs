X が動かない - 'xinit: not found'

/usr/X11R6/bin をあなたの path に入れてください。 root で 'set path = ( $path /usr/X11R6/bin )' を /etc/csh.cshrc に追加し、新たなシェルを起動してください。

startx が失敗する -'X connection to: 0.0 broken'

この場合、エラーメッセージを見るために、 
	startx >& ~/x.log
として起動してください。その後で、 view ~/x.log  

X が 'libXXXX.so.NN: No such file...'と言って落ちる

root で /etc/ld.so.conf を編集し、この行を追加してください: 
    /usr/X11R6/lib
同時にこの行も含んでいることも確認した方がいいでしょう: 
    /usr/pkg/lib
(追加プログラムを足すのに NetBSD パッケージを使っていますよね?)
その後 "ldconfig -v" を実行すると、出力の中から この形式の行を見つけられるでしょう: 'Adding /usr/X11R6/lib/libXIE.so.6.0'.

シェアードライブラリってなんですか?
ディスクとメモリを節約するために、ほとんどの NetBSD のポーティングでは 'シェアードライブラリ' を用います。各ライブラリのコピー (libXmu.so.6.0) は 複数のプログラム (xterm, fvwm2, etc) 用にメモリに読み込まれます。 標準のシステムのシェアードライブラリは /usr/lib にありますが、NetBSD は 追加のライブラリ (X のアプリケーション用など) のありかを知る必要が あります。そのためには '/etc/ld.so.conf' に追加をします。

コンソール表示 ('su'などの) が X の表示をこわす

システムは何かが起こったこと ('su'など) を表示するため コンソールに表示をします。 このコンソール表示をキャプチャするため xconsole か 'xterm -C' を起動 してください。

以下を有効にしてカーネルをコンパイルする必要があるかもしれません: 

	options  UCONSOLE  # allow anyone to steal the virtual console
詳しくは カーネルの作り方 を参照してください。

X のコンパイルに失敗する - unable to locate libXm

Xm は標準の X11 環境に入っていません。これは商用の GUI である Motif の一部です。

Motif のフリーな実装で Lesstif も利用できます。

X のコンパイルに失敗する - unable to locate xpm.h

xpm.h は X の PixMap ライブラリの一部で、カラーアイコンを作るため 多くのプログラムで使われています。 これは NetBSD パッケージ から利用可能です。
