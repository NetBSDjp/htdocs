NetBSD/i386 1.2

NetBSD/i386 1.2 は、 NetBSD/i386 の 5 回目の公式リリースです。

Charles Hannum と Frank van der Linden が、 1.2 リリースの時点での NetBSD/i386 のメンテナーです。

対応ハードウェア

NetBSD/i386 1.2 は、i386ファミリープロセッサーを搭載した ISA(ATバス)、EISA、 PCI、VLバスシステムで、数値演算コプロセッサーの有無に関わらず、動作します。 IBM PS/2 のような MCA システムには対応して*いません*。 最小の構成では 4MB の RAM と 50M のディスク容量が必要とされていますが、 今日このような最小構成のシステムを動かしているような人を私たちは知りません。 完全なシステムのインストールにはより多くのディスクスペースが必要ですし、 X の実行やシステムのコンパイルにはより多くの RAM を推奨します。 (RAM が 4MB でも確かに X の実行やコンパイルは可能ですが、速くはありません。 16 MB ほどの RAM もない場合は、 CPU を速くするより RAM を増やすほうが 重要なのです。) NetBSD/i386 1.2 で対応しているデバイスには、次のようなものがあります:

  • フロッピーコントローラー
    • MFM、 ESDI、 IDE 、および RLL ハードディスクコントローラー。
  • SCSI ホストアダプター:
    • Adaptec AHA-154xA, -B, -C, -CF
    • Adaptec AHA-174x
    • Adaptec AIC-6260 および AIC-6360 ベースのボード、 Adaptec AHA-152x および SoundBlaster SCSI ホストアダプターを含む。 (ブート ROM を持たないボードからのブートはできないことに注意; このため、 AHA-152x とこのチップを持つ マザーボードのみがブート可能のようです。)
    • Adaptec AHA-2x4x[U][W] カード、および、 AIC78X0 チップを使った オンボード PCI デザインのいくつか。このドライバーは、 AHA-2742 カードまたは AIC7770 もしくは AIC78[56]0 ベースの 組み込みデザインと複数のターゲットで使った場合に、 リソースの競合 (NetBSD の上位層の SCSI サブシステムが適切に割り当てない) による問題があることがわかっています。
    • Adaptec AHA-3940[U][W] カード [b]
    • Buslogic 54x (Adaptec AHA-154x クローン)
    • BusLogic 445, 74x, 9xx (新しい "FlashPoint" シリーズの BusLogic SCSI アダプターは含まない)
    • 下記のものを含む、 Seagate/Future Domain ISA SCSI アダプターカード
      • ST01/02
      • Future Domain TMC-885
      • Future Domain TMC-950
    • Symbios Logic (NCR) 53C8xx ベースの PCI SCSI ホストアダプター:
      • Acculogic PCIpport
      • ASUS SC-200 (マザーボードに NCR BIOS が必要)
      • ASUS SP3[G] マザーボードオンボード SCSI
      • DEC Celebris XL/590 オンボード SCSI
      • Lomas Data SCSI アダプター
      • NCR/SYM 8125 (および多くのクローン; 注意: これらのカードの一部には、 PCI 割り込み設定用のジャンパーがあります。これを INT A に設定してください!)
      • Promise DC540 (特に、 SYM 8125 のありふれた OEM モデル)
      • Tyan Yorktown
    • Ultrastor 14f、 34f、および (ことによると) 24f
    • Western Digital WD7000 SCSI ホストアダプター (ISA カードのみ)
  • MDA, CGA, VGA, SVGA および HGC ディスプレーアダプター。 (NetBSD/i386 で動作するディスプレーアダプターがすべて X でサポートされているわけでは ないことに注意。より詳しくは XFree86 FAQ を参照。)
  • シリアルポート:
    • 8250/16450 ベースのポート
    • 16550/16650/16750 ベースのポート
    • AST スタイル 4 ポートシリアルカード
    • BOCA 8 ポートシリアルカード
    • IBM PC-RT 4 ポートシリアルカード
    • シングルポート Hayes ESP シリアルカード
  • パラレルポート
  • イーサネットアダプター:
    • 下記のものを含む、 AMD LANCE および PCnet ベースの ISA イーサネットアダプター:
      • Novell NE1500T
      • Novell NE2100
      • Kingston 21xx
    • 下記のものを含む、 AMD PCnet ベースの PCI イーサネットアダプター:
      • Addtron AE-350
      • BOCALANcard/PCI
      • SVEC FD0455
      • X/Lan Add-On アダプター
      • IBM #13H9237 PCI イーサネットアダプター
    • AT&T StarLAN 10, EN100, StarLAN Fiber
    • 3COM 3c501
    • 3COM 3c503
    • 3COM 3c505
    • 3COM 3c507
    • 3COM 3c509, 3c579, 3c59X
    • 下記のものを含む、 Digital DC21x4x ベースの PCI イーサネットアダプター:
      • Cogent EM1X0, EM960 (別名 Adaptec ANA-69XX)
      • Cogent EM964 [b]
      • Cogent EM4XX [b]
      • Compex Readylink PCI
      • DANPEX EN-9400P3
      • Digital Celebris GL, GLST オンボードイーサネット
      • Digital (DEC) PCI Ethernet/Fast Ethernet アダプター (すべて)
      • JCIS Condor JC1260
      • Linksys PCI Fast Ethernet
      • SMC EtherPower 10, 10/100 (PCI のみ!)
      • SMC EtherPower^2 [b]
      • SVEC PN0455
      • SVEC FD1000-TP
      • Znyx ZX34X
    • BICC Isolan (最近はテストされていない)
    • Intel EtherExpress 16
    • Intel EtherExpress PRO/10
    • SMC/WD 8003, 8013 および SMC "Elite16" ISA ボード
    • SMC/WD 8216 (SMC "Elite16 Ultra" ISA ボード)
    • Novell NE1000, NE2000
  • FDDI アダプター:
    • Digital DEFPA PCI FDDI アダプター
    • Digital DEFEA EISA FDDI アダプター
  • テープドライブ:
    • ほとんどの SCSI テープドライブ
    • QIC-02 および QIC-36 フォーマットの (Archive および Wangtek 互換) テープドライブ
  • CD-ROM ドライブ:
    • IDE ではない Mitsumi CD-ROM ドライブ。 (注意: Mitsumi ドライバーデバイスプローブは、 同じアドレスに設定されたデバイスがあると問題を起こすことがわかっています)
    • ほとんどの SCSI CD-ROM ドライブ
  • マウス:
    • "Logitech" スタイルバスマウス
    • "Microsoft" スタイルバスマウス
    • "PS/2" スタイルマウス
    • シリアルマウス (カーネルサポート不要)
  • サウンドカード:
    • SoundBlaster
    • Gravis Ultrasound および Ultrasound Max (Gravis Ultrasound Plug&Play は除く)
    • Personal Sound System
    • Windows Sound System
    • ProAudio Spectrum

今のところ*未*対応だが、よく質問されるハードウェア:

  • AMD PCscsi SCSI ホストアダプター (PCnet-SCSI の PCnet 部はしっかり動作しますが)
  • APM 電源管理 -- APM 対応システムでは、これを無効にしてください!
  • Intel EtherExpress 100 Fast Ethernet アダプター。
  • マルチプロセッサー Pentium および Pentium Pro システム。 (プロセッサー 1 個だけを使ってならしっかり動くはずですが。)
  • NCR 5380 ベースの SCSI ホストアダプター。
  • PCI WD-7000 SCSI ホストアダプター。
  • PCMCIA ("PC カード") デバイス、小型の "IDE" ハードディスクを含む。
  • QIC-40 および QIC-80 テープドライブ。 (これらは、 フロッピーディスクコントローラーに接続するテープドライブです。)
将来的にはこれらのデバイスの多くにも対応したいと考えています。

もし NetBSD/i386 に興味をお持ちになったら、 より新しいバージョンをお使いください。
