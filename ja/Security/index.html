セキュリティーとNetBSD

NetBSDプロジェクトはセキュリティーに対して、 システムの他の部分にするのと同じアプローチを採用しています: 解決でありハックではありません。 NetBSDのセキュリティー問題は、NetBSDセキュリティーオフィサー、 およびNetBSDセキュリティー警告チームによって取り扱われます。 NetBSDセキュリティー警告チームは、 新たにレポートされたセキュリティー問題に応じて、 調査はもちろん、文章化、コードの更新を行います。 またチームは、潜在的なセキュリティー問題を捜し出して削除するために、 定期的なコードの検査を行っています。

セキュリティーの連絡

NetBSDプロジェクトは、二つのセキュリティーに関する連絡ポイントを持っています:

  • tech-security メーリングリストは、NetBSDのセキュリティーに関する問題を議論するための 公開されたフォーラムです。
  • security-alert@netbsd.org に電子メールを送ることによって、 直接NetBSDプロジェクトにセキュリティー問題について 問い合わせることができます。

セキュリティー問題のレポート

NetBSDにおけるセキュリティー問題をレポートする方法として、 NetBSDセキュリティー警告チームに連絡する方法と、 障害報告フォーム もしくはNetBSDシステム上の send-pr(1) プログラムを 使用して標準的なNetBSD障害レポートを送る方法があります。

機密扱いの情報は、NetBSDセキュリティーオフィサーの PGPキーを使って、 暗号化するべきです。PGPについての情報は Network Associatesから得ることができます。

セキュリティーパッチ

すべてのNetBSD セキュリティーパッチは、NetBSDプロジェクトの FTP serverの misc/security/patches/ディレクトリーから入手可能です。

NetBSD リリース への、および NetBSD current への更新の詳細は以下にあります。

  • NetBSD 1.3
  • NetBSD 1.3.1
  • NetBSD 1.3.2
  • NetBSD 1.3.3

セキュリティーリソース

多くのセキュリティー勧告、および他のセキュリティーリソースは、 これらのサイトでオンラインに利用できます:

  • NetBSDプロジェクト 勧告アーカイブ
  • Computer Emergency Response Team
  • Australian Computer Emergency Response Team
  • Forum of Incident Response and Security Teams
  • SecurityPortal.com - 破壊に関するニュース、調査、およびセキュリティー分野の検索エンジン
  • Secure Networks Inc
  • The FreeBSD Project Security Guide
