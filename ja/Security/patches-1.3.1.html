NetBSD 1.3.1 Security Patches

The NetBSD 1.3 branch is no longer being actively maintained; security patches are not being back-ported to NetBSD 1.3; as a result, this page is no longer actively being maintained. This page is of historic interest only.

Users should upgrade to NetBSD 1.4.3 or NetBSD 1.5

Below is the list of patches applicable to the NetBSD 1.3.1 release:

  • BIND vulnerabilities which are described in NetBSD Security Advisory SA1998-001
  • xterm(1) and Xaw library vulnerabilities which are described in NetBSD Security Advisory SA1998-002
  • mmap(2) vulnerabilities which are described in NetBSD Security Advisory SA1998-003
  • at(1) vulnerabilities which are described in NetBSD Security Advisory SA1998-004
  • mmap(2) vulnerabilities which are described in NetBSD Security Advisory SA1998-005
  • select(2)/accept(2) race condition in TCP servers which is described in NetBSD Security Advisory SA1999-001
  • traceroute(1) problems which is described in NetBSD Security Advisory SA1999-004
  • umapfs problems which is described in NetBSD Security Advisory SA1999-006
  • mount(2) problems which is described in NetBSD Security Advisory SA1999-007
  • vfs_lookup locking problems which is described in NetBSD Security Advisory SA1999-008
  • i386 SVR4 compatibility problems. which is described in NetBSD Security Advisory SA1999-009
  • ARP problems. which is described in NetBSD Security Advisory SA1999-010
  • A description and resolution procedure for - buffer overrun in libc DNS resolver can be found in NetBSD Security Advisory SA2002-006
