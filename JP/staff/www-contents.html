WWWコンテンツの管理

  • 日本独自コンテンツとwww.netbsd.orgの日本語訳は CVS(CVSの使い方)で管理する。
  • モジュールは htdocs。
  • 更新時には www-changes-ja@jp.netbsd.org に更新情報が流れる。

WWWコンテンツの配置

  • http://www.netbsd.org/の ミラーはhttp://www.jp.netbsd.org/ 以下で参照できる。但しhttp://www.jp.netbsd.org/ は、 http://www.jp.netbsd.org/ja/JP/index.html にリダイレクトされる。
  • 日本語訳は、オリジナルと同じ構成で htdocs/ja/ 以下に作成する。
    http://www.jp.netbsd.org/ja/ 以下で参照できる
  • 日本語の独自コンテンツは htdocs/JP/ 以下に作成する。
    http://www.jp.netbsd.org/ja/JP/ 以下で参照できる

www.netbsd.org 日本語訳

  • 翻訳する前に htdocs/JP/staff/translation-assign.txt の 翻訳するファイルの担当者欄に名前を記入すること。 このとき、査読者の欄は空欄とする。 アカウントがない場合は、 www-changes-ja@jp.netbsd.orgにて宣言すること。

  • 翻訳したファイル、および修正/提案などは直接commitするか、 www-changes-ja@jp.netbsd.orgに流すこと。
    翻訳者が、訳したファイルを最初に commit する時には、 translation-assign.txt の査読者 の欄を、空欄から NOT-YET に変更し、スクリプト ./www-pending.sh を実行しておくこと。 (www-pending.sh が生成した www-pending.html も commit する)
  • 査読を commit する時には、 translation-assign.txt の査読者の欄に、カンマで区切って名前を 記入し、スクリプト ./www-pending.sh を実行しておくこと。 (www-pending.sh が生成した www-pending.html も commit する)

  • CVSリポジトリにアクセスできない人の翻訳ファイルや修正点などは、 以下の担当者 がリポジトリに反映する。ただし、www-changes-ja を講読していて、 かつリポジトリにアクセスできる人は、誰でも反映して良い。 (担当者は、誰も反映しなかった場合の最後の砦という意味に過ぎないので注意)
    • sakamoto@jp.netbsd.org

  • 翻訳した文書中の www@NetBSD.ORG は www@JP.NetBSD.ORG に変更すること。
  • 翻訳時には、$NetBSD: www-contents.html,v 1.5 1999/01/26 05:55:35 sakamoto Exp $ の tag と、 TNF の Copyright notice を削らないこと。
  • 翻訳の際は、 htdocs/JP/staff/translation-terms.txt に 極力統一すること(特に術語[term])。

  • 翻訳でのCVS手続き
    新規およびオリジナルファイル更新時、 どちらとも以下の手続きで行う。
    1. リポジトリを一時的にバックアップ
      tar zcf ~/ja-bak.tar.gz -C /cvsroot htdocs/ja
    2. 翻訳したいオリジナルファイルだけをカレントディレクトリに置く
    3. CVSROOT,CVS_RSHなどを設定後、
      cvs import -I \! -ko DIR VENDOR RELEASE
      で登録

      DIR オリジナルのディレクトリパスの頭に ja/を付ける
      (http://www.netbsd.org/hoge/piyo.htmlの場合は htdocs/ja/hoge になります)。
      VENDOR TNF を指定(The NetBSD Foundation)。
      RELEASE date-1998-01-10 というように オリジナルのファイルを www.*.netbsd.org から取得した日時ないし $NetBSD: www-contents.html,v 1.5 1999/01/26 05:55:35 sakamoto Exp $ の日時を指定。

      例: cvs import -I \! -ko htdocs/ja/hoge TNF date-1998-01-10

    4. checkout後、オリジナルファイルの上に和訳を上書きしcommit。
      この時、translation-assign.txt の査読者の欄を、NOT-YET に変更し、スクリプト ./www-pending.sh を 実行しておくこと。(translation-assign.txt および、www-pending.sh が生成した www-pending.html も commit すること)
